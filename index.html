Tentang Github

GitHub adalah platform berbasis web untuk mengelola proyek dan versi kode menggunakan Git, sebuah sistem kontrol versi yang populer di kalangan developer. GitHub digunakan oleh individu maupun tim untuk menyimpan, melacak perubahan, dan berkolaborasi dalam proyek perangkat lunak.

Fitur Utama Github

  • Repository (Repo): Tempat menyimpan semua file proyek, termasuk histori perubahan.
  • Commit: Mencatat perubahan yang dilakukan pada file. Commit biasanya punya pesan deskriptif.
  • Branch: Versi paralel dari proyek, biasanya digunakan untuk mengembangkan fitur baru tanpa mengganggu branch utama (main atau master).
  • Pull Request (PR): Permintaan untuk menggabungkan perubahan dari satu branch ke branch lainnya. Bisa didiskusikan dan direview sebelum digabung.
  • Issue Tracker: Sistem pelaporan bug, permintaan fitur, atau diskusi lainnya.
  • Actions: Fitur CI/CD GitHub yang bisa otomatis menjalankan test, build, atau deployment.
  • Wiki & Pages:
    • Wiki: Untuk dokumentasi proyek.
    • Pages: Untuk membuat website statis dari repo.

Jenis Repository

  • Public: Bisa dilihat semua orang.
  • Private: Hanya bisa diakses oleh pemilik dan kolaborator.

Alur Dasar Penggunaan GitHub

  1. Buat akun di github.com
  2. Buat repository baru
  3. Clone ke komputer lokal
  4. Tambahkan dan ubah file
  5. Commit perubahan
  6. Push ke GitHub

Kelebihan GitHub

  • Kolaborasi real-time dengan tim
  • Backup otomatis di cloud
  • Riwayat lengkap perubahan
  • Mendukung proyek open-source dan pribadi
